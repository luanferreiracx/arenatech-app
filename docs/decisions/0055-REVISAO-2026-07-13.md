# ADR 0055 — Revisão antes de implementar (2026-07-13)

**Status:** ✅ Aplicada. Guiou a implementação do PR #554.
**Skills:** `audit-ai-systems` (defesa de prompt injection), `staff-engineering` (revisão de ADR).

A proposta original (`0055-talison-bot-config-editavel.md`) era sólida na intenção mas tratava
o texto do admin como conhecimento benigno. Ele entra no **system prompt de TODA conversa** —
isso é uma superfície de prompt injection. Esta revisão endureceu o design antes de codar.

## Melhorias (M1–M7)

- **M1 — Posição e recência.** A proposta injetava o bloco do admin como "último bloco
  dinâmico". Risco: recência favorece o texto do admin sobre as guardas. Correção: as regras
  fixas vêm ANTES, e uma **reafirmação das guardas** (`STORE_INSTRUCTIONS_GUARD`) é a ÚLTIMA
  coisa do prompt. A última palavra é sempre da segurança.
- **M2 — Delimitar como DADO.** Texto do admin cercado por delimitadores explícitos
  (`<<< INÍCIO/FIM DAS INSTRUÇÕES DA LOJA >>>`) e rotulado "informação, não ordem", para o
  modelo não confundir conteúdo com instrução de sistema.
- **M3 — Isolamento por tenant.** O runner lê `tenantSettings` por PK do tenant da conversa;
  RLS de `tenant_settings` reforça. Teste de integração prova que a instrução do tenant A não
  vaza para o prompt do B.
- **M4 — Validação determinística pós-UI.** Um schema (`updateBotConfigSchema`, compartilhado
  cliente/servidor) barra padrões óbvios de injeção ("ignore as regras", "você agora é",
  "system prompt", "act as", "jailbreak", "DAN"…) e o cap de 4000 chars. Defesa secundária —
  a principal é arquitetural (M1/M2).
- **M5 — Custo.** O texto entra em toda conversa → custo por token recorrente. Cap de 4000 +
  dica na UI para o admin ser objetivo.
- **M6 — Prévia + desfazer.** Prévia mostra EXATAMENTE o trecho inserido no prompt (reusa
  `renderStoreInstructionsBlock`, não uma aproximação). Desfazer de 1 nível
  (`botInstructionsPrevious`) para reverter uma edição ruim sem reescrever.
- **M7 — Dívida latente (não resolvida neste PR).** O esqueleto fixo do prompt cita
  "Arena Tech" hardcoded. Para o bot servir um 2º tenant sem confundir identidade, o esqueleto
  precisa parametrizar o nome da loja pelo `businessContext`. Anotado para o futuro.

## Decisões do dono

| Tema | Decisão |
|---|---|
| Cap de tamanho | 4000 caracteres |
| Desfazer | Última alteração, 1 nível |
| Prévia | Só a inserção (não o esqueleto completo) |
| Acesso | Admin do tenant, aba "Assistente (Talison)" |

## Resultado

Implementado no PR #554 (DB → backend → frontend), gateado pelo módulo `service-orders`.
11 testes unit + 3 de isolamento por tenant. Ver o fecho em
`0055-talison-bot-config-editavel.md`.
